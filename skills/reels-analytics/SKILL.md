# מודול אנליטיקס — ניתוח ביצועי רילס

| מקור | מה צריך |
|------|---------|
| **CSV מ-Meta** | קובץ ייצוא של Instagram Insights |

| **ידני** | המשתמש מדביק טבלה |

### שדות נדרשים (מינימום)

| שדה | תיאור |
|-----|--------|
| תאריך פרסום | מתי יצא |
| צפיות (views) | מספר צפיות |

## זרימת עבודה — 6 שלבים

### שלב 1 — שליפה ומיפוי

1. שלוף את כל הנתונים מהמקור
2. נקה: הסר ערכים חסרים, תקן פורמט תאריכים
3. חשב מדדים נגזרים:
   - **Engagement Rate** = (לייקים + תגובות + שיתופים + שמירות) / צפיות × 100
   - **Save Rate** = שמירות / צפיות × 100
   - **Share Rate** = שיתופים / צפיות × 100
   - **Virality Score** = שיתופים / (לייקים + תגובות) × 100

---

### שלב 2 — Top/Bottom Analysis

**Top 5:**
- 5 הרילסים עם הצפיות הגבוהות ביותר
- 5 הרילסים עם ה-Engagement Rate הגבוה ביותר
- 5 הרילסים עם ה-Save Rate הגבוה ביותר

**Bottom 5:**
- 5 הרילסים עם הביצועים הנמוכים ביותר

**לכל אחד — ציין:**
- מה הנושא/הוק
- מה האורך
- מה ייחודי (או מה חסר)

---

### שלב 3 — ניתוח מתאמים

חפש קורלציות בין:

| משתנה | לבדוק מול |
|-------|-----------|
| אורך סרטון | צפיות, engagement |
| שעת פרסום | צפיות |
| סוג הוק (שאלה/הצהרה/סטוריטלינג) | save rate, shares |
| נושא/קטגוריה | ביצוע כללי |
| שימוש בטקסט על המסך | engagement |

הצג: "יש מתאם / אין מתאם / מתאם הפוך" + דוגמאות.

---

### שלב 4 — בנצ'מרק

השווה ביצועים ל:
- **ממוצע עצמי** — מול 30/60/90 ימים אחרונים
- **מגמה** — עולה / יורד / יציב (לפי חודשים)

הצג בטבלה:

| מדד | ממוצע 30 יום | ממוצע 60 יום | ממוצע 90 יום | מגמה |
|-----|-------------|-------------|-------------|------|
| צפיות | | | | ↑/↓/→ |
| Engagement | | | | |
| Saves | | | | |
| Shares | | | | |

---

### שלב 5 — סינתזה ותובנות

מתוך ניתוח שלבים 2-4, חלץ:

1. **3 דפוסי הצלחה** — מה משותף לרילסים שעבדו
2. **3 דפוסי כישלון** — מה משותף לרילסים שלא עבדו
3. **הפתעות** — רילסים שהביצוע שלהם לא תואם את הציפיות (למעלה או למטה)
4. **המלצה #1** — השינוי הכי משמעותי שכדאי לעשות

## כללי עבודה

1. **נתונים קודם** — אל תעשה ניתוח בלי נתונים. אם אין מספיק — בקש
2. **מינימום 10 רילסים** — פחות מזה = אין מספיק לניתוח סטטיסטי. ציין זאת
3. **אל תמציא מספרים** — אם מדד חסר, אמור "חסר" ולא להמציא
4. **המלצות פרקטיות** — כל תובנה חייבת להסתיים ב"מה לעשות"
5. **השוואה לעצמך** — אל תשווה לבנצ'מרק "כללי" אלא אם המשתמש מבקש
