# מדריכים — ידע קורס אִימפּרוּב

> מסמך ידע קבוע. ידע על מבנה סוכנים, מבנה קבצי קלוד, מבנה הריפו בשיטת אִימפּרוּב, ודאטהבייס המדריכים.
> כשהמשתמש שואל על מושג מהקורס — אני עונה מהידע הזה + מציע לחפש עוד בדאטהבייס.

**איך להפנות:** כשמשתמש שואל על מושג שיש עליו מדריך — אני אומר:

אני נותן **נושא לחיפוש**, לא שם מדריך ספציפי. הדאטהבייס מתעדכן וייתכן שיש גרסאות חדשות.

**מתי להפנות:**

- כשהמשתמש רוצה לחזור ולבדוק הגדרה

## מדריך 1: איך נראה סוכן שותף מבפנים

**מטאפורה:** סוכן שותף = אדם.

כל בן אדם בנוי משלושה דברים:
- אופי — מי הוא, איך הוא חושב, איך הוא מדבר
- ידע — מה הוא יודע על העולם, על עצמו, על אנשים
- מיומנויות — מה הוא יודע לעשות

סוכן שותף בנוי באותה הדרך. בתוך התיקייה שלו יש 3 חלקים:

```
[שם-הסוכן]/
├── system-prompt.md          ← 🎭 האופי
├── master-library/            ← 📚 הידע
└── skills/                    ← 🛠️ המיומנויות
```

**שכבה 1 — האופי** (`system-prompt.md`):
- מי הסוכן — תפקידו, השם שלו
- איך הוא חושב — סגנון החשיבה, מה חשוב לו
- איך הוא מדבר — טון, סגנון, רמת פורמליות
- מה חשוב לו — ערכים, עקרונות פעולה
- נטען ראשון בכל שיחה. קובע מי הסוכן.

**שכבה 2 — הידע** (`master-library/`):
- ידע על המשתמש (מי הוא, מה העסק, מה המטרות)
- ידע על העולם הרלוונטי (תחום, קהל, מתחרים)
- ידע על תהליכים (איך עושים X, מה הסטנדרט של Y)
- הסוכן קורא מהתיקייה הזאת בכל שיחה ומשתמש בה כרקע.

**שכבה 3 — המיומנויות** (`skills/`):
- כל מיומנות יושבת בתיקייה משלה
- סוכן טוען מיומנות כשמזהה שצריך אותה
- סוכן יכול להתחיל **בלי** מיומנויות — רק אופי וידע. מיומנויות מתווספות בהדרגה.

**שכבה 4 אופציונלית — זיכרון** (`memory.md`):
- ידע = קבוע, נשמר במסמכי ה-master-library
- זיכרון = דברים שהסוכן רואה במהלך השיחות ושהמשתמש מבקש לזכור

## מדריך 2: איך נראה סוכן תהליך מבפנים

**מטאפורה:** סוכן תהליך = מכונה אוטומטית (כמו רומבה).

אין לו אופי, אין לו שיחה. הוא יושב במחשב, רץ ברקע, ומבצע משימה **אחת ספציפית** — שוב ושוב, באותה דרך, בלי שתצטרך להגיד לו כלום.

**המבנה הבסיסי:**

```
[שם-המכונה]/
├── [קובץ קוד]            ← ⚙️ המנוע
├── [קובץ הגדרות]        ← 🎛️ לוח הבקרה
├── [קובץ הפעלה]         ← 🔘 כפתור ההפעלה
└── [קובץ הלוג]            ← 📊 המסכים
```

**חלק 1 — המנוע (מה היא עושה):**
- קבצי הקוד הראשיים — "המתכון" של המכונה
- לא צריך להבין מה כתוב בתוכם — מספיק לדעת מה תפקיד כל קובץ
- משתנה ממכונה אחת למכונה אחרת

**חלק 2 — לוח הבקרה (איך מגדירים אותה):**
- קבצי הגדרות (`.env`, `config.json`)
- מפתחות API — כדי שתוכל לדבר עם שירותים חיצוניים
- נתיבים — איפה למצוא את החומר, איפה לשמור את התוצאה
- העדפות — לאן לשלוח מייל, מי המקבל, באיזה סגנון
- זה החלק שמשנים כשרוצים לשנות איך המכונה מתנהגת — בלי לגעת בקוד
- **חשוב:** מפתחות API הם סודיים — תמיד בקובץ `.env`, לא מועלים ל-GitHub

**חלק 3 — כפתור ההפעלה (מתי היא מתחילה):**
- כפתור ידני — אתה לוחץ, היא מתחילה
- שעון מתוכנת — מופעלת בזמן קבוע (כל שעה / יום / שבוע)
- חיישן — מופעלת כשמשהו קורה (קובץ חדש, מייל, אירוע)

**חלק 4 — המסכים (איך עוקבים):**
- לוגים — יומן הפעולות ("התחלתי בשעה X. סיימתי שלב 1. נכשלתי בשלב 2.")
- State — הזיכרון של המכונה ("כבר עיבדתי את ההקלטה הזאת, אל תעבדי שוב")
- כשמכונה רצה ברקע — הלוגים וה-state הם הדרך היחידה לדעת שהכל תקין

## מדריך 3: מפת קבצי קלוד — איפה הכל יושב במחשב שלך

**עיקרון:** כשקלוד יוצר קובץ — זה קובץ אמיתי על המחשב שלך, לא משהו שנעלם בסוף השיחה.

**איפה קלוד שומר:**
בתיקייה שנבחרת ב-Cowork. כשפותחים פרויקט חדש — בוחרים תיקייה. כל קובץ שקלוד יוצר נשמר שם. טיפ: תיקייה ייעודית לכל פרויקט.

**מבנה תיקיית פרויקט עם קלוד:**

```
my-project/                 ← התיקייה שלך
│
├── CLAUDE.md               ← הכללים (נטענים בכל שיחה)
├── CLAUDE.local.md         ← אישיים (נשארים רק במחשב)
│
├── .claude/                ← תיקיית קלוד (מוסתרת)
│   ├── skills/             ← הסקילים
│   └── settings.json       ← הרשאות: מה קלוד מורשה לעשות לבד
│
└── (הקבצים שלכם כאן)
```

**הבדל: CLAUDE.md לעומת סקיל:**
- `CLAUDE.md` — הוראות קבועות שתמיד נטענות. קובע "מי אתה ואיך אתה עובד"
- סקיל — נטען רק כשקלוד מזהה שהמשימה מתאימה לו. נותן "איך לעשות את X"
- השניים משלימים זה את זה

**הערות טכניות:**
- תיקיית `.claude/` מוסתרת — ב-Mac: `Cmd+Shift+.` בפיינדר להציג; ב-Windows: View → Show → Hidden items
- כל פרויקט בתיקייה משלו, בוחרים ב-Cowork לפני שמתחילים

## מדריך 4: מבנה הריפו בשיטת אִימפּרוּב

**מטאפורה:** הריפו = ארון בגדים / משרד. בלי מבנה — בלגן. עם מבנה מההתחלה — הכל נגיש.

**ריפו (Repository):** תיקייה אחת גדולה במחשב, מסונכרנת עם GitHub (גיבוי בענן).

**4 אזורים בתוך הריפו:**

**1. הספרייה הפרטית** (`master-library/`):
- ידע עליך — מי אתה, מה העסק, מה אתה רוצה להשיג, איך אתה אוהב לעבוד
- **משותפת לכל הסוכנים** — כל סוכן שתבנה יקרא ממנה
- תמיד מוסיפים, אף פעם לא מוחקים

**2. שולחן העבודה** (`agents/in-progress/`):
- כאן בונים סוכן חדש
- בזמן הבנייה: הסוכן בפיתוח, הציטוטים, ההחלטות, הסקיצות
- ב-8 המפגשים עם בְּנָיָה, Agent 2 נבנה כאן

**3. הארכיון** (`agents/completed/`):
- כשסוכן סיים להיבנות, **קבצי הבנייה שלו** (ההיסטוריה) נשמרים כאן
- שווה לחזור ולראות איך נבנה — במיוחד אם בונים משהו דומה בעתיד

**4. המדף של הסוכנים הפעילים** (`agents/ready/`):
- כאן גרים הסוכנים הפעילים — אלה שמוכנים, רצים, ועובדים
- רק "קבצי התפעול" — מה שמפעיל אותם — נמצא כאן. בלי היסטוריה
- הסוכן הפעיל צריך להיות **רזה** — לא לסחוב 8 מפגשים של היסטוריה

**מחזור חיים של סוכן:**
```
בנייה (in-progress) → גמר (in-progress → split)
                              ↓                 ↓
                     קבצי בנייה          קבצי תפעול
                     → completed/         → ready/
```
אחרי מפגש 8, הסוכן עובר פיצול:
- קבצי הבנייה → ארכיון
- קבצי התפעול → מדף פעילים
- שולחן העבודה מתפנה — מוכן לסוכן הבא

**מבנה הריפו אחרי הקורס:**
```
[שם-הריפו]/
├── master-library/         ← הידע עליך (משותף לכל הסוכנים)
├── agents/
│   ├── in-progress/        ← ריק (שולחן פנוי)
│   ├── completed/
│   │   └── agent-2-dev/    ← ההיסטוריה של בניית Agent 2
│   └── ready/
│       └── agent-2/        ← Agent 2 חי ועובד
├── README.md               ← קובץ הסבר על הריפו
└── CLAUDE.md               ← קובץ זיכרון של הריפו
```
