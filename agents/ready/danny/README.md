# דני — עוזר פגישות

> סוכן דמו לצורך הדגמה בשיעור. לא סוכן אמיתי — מיועד להראות מבנה.

## מה דני עושה
דני הוא **סוכן שותף** שעוזר לבעל עסק קטן לנהל פגישות:
- מסכם פגישות לתבנית מסודרת
- יוצר כרטיס לקוח חדש ב-Notion אחרי פגישה ראשונה

## המבנה שלו

```
danny-meeting-agent/
├── system-prompt.md          ← 🎭 האופי — מי דני ואיך הוא עובד
├── skills/                   ← 🛠️ המיומנויות
│   ├── meeting-summary/      ← סקיל חופשי: מסכם פגישה (שיקול דעת)
│   │   └── SKILL.md
│   └── client-card/          ← סקיל מסודר: יוצר כרטיס לקוח (צעדים קבועים)
│       └── SKILL.md
└── knowledge/                ← 📚 הידע
    └── notion-crm-guide.md   ← מה דני יודע על ה-CRM ב-Notion
```

## למה דני

דני הוא דוגמה פשוטה שכל בעל עסק מכיר:
- פגישות = חלק מהיומיום של כולם
- סיכום + תיעוד = משימה שכולם שוכחים/עוקפים
- Notion = מערכת שכולם בקורס מכירים

## שני סוגי סקילים

| סוג | דוגמה בדני | מה ההבדל |
|-----|-----------|----------|
| **חופשי** | meeting-summary | הסוכן מפעיל שיקול דעת — מחליט מה חשוב ומה לא |
| **מסודר** | client-card | הסוכן עוקב אחרי צעדים קבועים — שאלה 1, שאלה 2, שמור |
