# ניתוח ביצועי תוכן — מתודולוגיה שיטתית

## מה זה עושה
מערכת ניתוח 6-שלבית שלוקחת נתוני ביצועים של תוכן (רילסים, פוסטים, סרטונים) ומפיקה דוח תובנות מעמיק — מה עובד, למה, ומה לשנות.

## למה זה שימושי
במקום "להרגיש" מה עובד — יש לך שיטה מובנית שמנתחת בכמה שכבות: מספרים, hooks, תיאורים, ותמלולים. התוצאה: חוקי הצלחה ברורים שאפשר ליישם מחר.

## מה צריך
- [ ] מקור נתונים (Notion DB / Google Sheets / CSV) עם נתוני ביצועים
- [ ] AI assistant שמריץ את הניתוח (Claude / ChatGPT)
- [ ] תמלולים של התוכן (לניתוח עומק)

## איך מריצים

זו לא סקריפט — זו **מתודולוגיה** שנותנים ל-AI לבצע. אפשר להדביק אותה כהוראות ל-Claude/ChatGPT.

### הזרימה
```
נתוני ביצועים (DB / CSV)
       ↓
[0] הכנה — ניקוי נתונים + חישוב מטריקות נגזרות
       ↓
[1] ניתוח כמותי — דירוג, מתאמים, Top/Bottom
       ↓
[2] ניתוח Hooks — מה עוצר גלילה?
       ↓
[3] ניתוח תיאורים — מה מביא Reach?
       ↓
[4] ניתוח תמלולים — מה עובד בעומק?
       ↓
[5] סינתזה — חוקי הצלחה + אנטי-פטרנים + דלתא
```

### שימוש בפועל
```
1. העתיקו את analysis-methodology.md כ-system prompt
2. תנו ל-AI גישה לנתונים (CSV / Notion MCP / Google Sheets)
3. בקשו: "הרץ ניתוח לפי המתודולוגיה"
4. קבלו דוח תובנות מלא
```

## איך להתאים אישית

### מקור הנתונים
המתודולוגיה עובדת עם כל מקור נתונים. התאימו את שדות ה-DB ב-`metrics-definitions.md` לשדות שלכם.

### סוג התוכן
למרות שהדוגמה כאן היא לרילסים — אותה שיטה עובדת על:
- פוסטים בלינקדאין
- סרטוני יוטיוב
- פרקי פודקאסט
- מיילים שיווקיים
- כל תוכן עם נתוני ביצועים

### רמות ביטחון
כל תובנה מסומנת ברמת ביטחון:
- **חזק** — חוזר ב-3+ פריטים
- **בינוני** — חוזר ב-2
- **השערה** — פעם אחת, צריך אימות
